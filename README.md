# Flask Service（医疗助手智能服务层）

本模块是 MedicalAssistant 的智能能力服务层，负责承接大模型、检索增强、预测分析与工具编排能力，并通过统一 API 向上层业务系统提供智能服务。

---

## 模块定位

`flask-service` 的核心定位是：  
**将医疗场景中的“理解、检索、推理、生成、预测”能力封装为可调用服务**，与主业务后端协同工作，形成完整的医疗智能助手能力闭环。

它主要解决以下问题：

- 将复杂 AI 能力（Agent/RAG/预测）标准化为稳定接口
- 降低业务系统直接对接模型与知识库的复杂度
- 为医疗问答、健康建议、风险识别等场景提供智能支持
- 在“可扩展工具链”架构下持续接入新能力

---

## 核心能力概览

### 1. Agent 智能编排能力

位于 `app/services/agent/`。

- 通过 Orchestrator 对请求进行任务分解与流程调度
- 基于路由策略将问题分派到不同工具（如检索、业务接口、推理组件）
- 支持工具注册机制，便于后续扩展新工具类型
- 提供多步骤协同能力，而非单轮静态问答

**价值**：让系统从“单点回答”升级为“可规划、可调度、可扩展”的智能体执行模式。

---

### 2. RAG 检索增强能力

位于 `app/services/rag_service.py` 与 `app/services/rag_light/`。

- 将用户问题与医疗知识库进行语义匹配
- 支持图谱/向量/规则等多层检索与结果重排
- 将检索证据与生成回答结合，降低模型幻觉风险
- 面向医疗知识问答提供更可靠的知识支撑

**价值**：让回答“有依据、可追溯、领域相关性更高”。

---

### 3. 预测与分析能力

位于 `app/services/predict/`。

- 集成医学文本语义模型（如 ClinicalBERT 相关资源）
- 支持对输入健康数据或文本进行风险倾向分析
- 可扩展为分层预警、趋势判断与辅助决策能力

**价值**：从“回答问题”延伸到“识别风险、辅助判断”。

---

### 4. OCR 与非结构化信息接入

位于 `app/routes/ocr.py` 及相关服务。

- 面向医疗场景中的图片/文档文本提取需求
- 将非结构化信息转为可分析、可检索的数据输入
- 为后续 Agent 推理和 RAG 检索提供结构化上下文

**价值**：打通影像/报告等非结构化数据到智能分析链路。

---

### 5. 联邦学习相关能力接口

位于 `app/services/federated/`。

- 提供联邦场景下的客户端协作与隐私机制支持
- 关注跨参与方协同时的数据安全与参数交互逻辑
- 为后续隐私保护医疗建模预留服务化能力

**价值**：支持“数据不出域”的协同智能演进方向。

---

## 服务设计特点

- **能力解耦**：路由层、服务层、工具层分离，职责清晰  
- **可扩展**：工具注册与模块化目录设计，便于新增能力  
- **多能力融合**：Agent + RAG + Predict + OCR 可组合调用  
- **面向医疗场景**：围绕健康问答、知识检索、风险分析构建  
- **工程化落地**：以 API 形式输出能力，便于与主系统集成

---

## 在整体系统中的角色

在 MedicalAssistant 架构中，`flask-service` 不是传统业务后端的替代，而是：

- 作为**AI 能力中台**
- 对接模型、知识库、图谱和工具链
- 将复杂智能流程封装为稳定服务接口
- 向上支撑用户问答、健康咨询、辅助分析等业务功能

它连接了“医疗数据”和“智能决策”之间的关键一层。

---

## 适用场景（示例）

- 医疗知识问答与用药信息检索  
- 健康风险提示与文本级预测分析  
- 报告/图片信息抽取后的一体化智能解读  
- 多工具协同的复杂健康咨询流程  

---

## 总结

`flask-service` 的本质是一个**面向医疗智能场景的能力编排与服务输出层**。  
它通过 Agent、RAG、预测与 OCR 等模块协同，把分散的 AI 技术能力组织成可落地、可扩展、可集成的智能服务体系。
